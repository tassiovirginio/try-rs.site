deserve a place.

A lightning-fast TUI to manage temporary projects. Create, explore, and clone repositories instantly without cluttering your system.

cargo install try-rs
Get Started
tassiovirginio@rust-box: ~/projects/try-rs
Real TUI Demo

CLI Usage

Launch the interactive TUI interface:

try

Create a new temporary workspace with a specific name:

try <workspace_name>

Clone a repository directly into a temporary workspace:

try <github_url>

Everything try-rs can do

A robust feature set built for speed and organization.

Instant Workspaces

Generate isolated environments in milliseconds. No more cluttering your /tmp or home folder manually.

Fuzzy Filtering

Start typing anywhere in the TUI. Built-in fuzzy matching helps you find that project from two weeks ago instantly.

Git Integration

Provide a GitHub/GitLab URL as an argument. try-rs clones it into a new dated folder and drops you in the shell.

Interactive TUI

A beautiful Ratatui-powered terminal interface. Browse your experiments with zero latency and full visual feedback.

Safe Cleanup

Delete old workspaces directly from the interface with Ctrl+D. Includes a safety confirmation popup.

Editor Launch

Open your favorite IDE (VSCode, Neovim, Helix) instantly in the selected workspace using the Ctrl+E shortcut.

Dated Hierarchy

Automatically organizes folders using a YYYY-MM-DD-name format, making chronological navigation effortless.

Cross-platform

Native support for Linux, macOS, and Windows. Uses OS-standard configuration paths for a native feel everywhere.

Flexible Config

Configure your preferred shell, custom storage paths, and UI behavior via a simple TOML file or environment variables.

Configuration

Customize your experience by creating a config.toml file. The location depends on your operating system:

Linux
~/.config/try-rs/config.toml
macOS
~/Library/Application Support/try-rs/config.toml
Windows
%AppData%\try-rs\config.toml

Config File Example

# config.toml
tries_path = "~/Development/playground"
editor = "code" # Optional: code, nvim, hx, etc.

[colors]
title_try = "Magenta"
title_rs = "White"
search_box = "Green"
list_date = "Blue"
list_highlight_bg = "Magenta"
list_highlight_fg = "Black"
help_text = "DarkGray"
status_message = "Yellow"
popup_bg = "DarkGray"
popup_text = "LightRed"

Environment Variables

TRY_PATH

Overrides the default storage path.

export TRY_PATH="/home/user/tmp"
TRY_CONFIG

Specify a custom path for the config file.

export TRY_CONFIG="~/.try.toml"

Install Now

Installation Command
$ cargo install try-rs

Requirements: cargo (Rust toolchain) installed on your system.
