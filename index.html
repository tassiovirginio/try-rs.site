CLI Commands

You can also bypass the UI and use direct commands:

Command Description
try-rs Opens the TUI
try-rs <name> Create (or jump to) a named experiment
try-rs <url> Clones a repository into a dated folder
try-rs <url> <name> Clones a repository into a specific folder name
try-rs -s <url> Shallow clone (depth 1)
try-rs --setup <shell> Setup shell integration (fish, zsh, bash, power-shell, nu-shell)
try-rs --version Show application version
try-rs --help Show help message

Everything try-rs can do

A robust feature set built for speed and organization.

Instant Workspaces

Generate isolated environments in milliseconds. No more cluttering your /tmp or home folder manually.

Fuzzy Filtering

Start typing anywhere in the TUI. Built-in fuzzy matching helps you find that project from two weeks ago instantly.

Git Integration

Provide a GitHub/GitLab URL as an argument. try-rs clones it into a new dated folder and drops you in the shell.

Interactive TUI

A beautiful Ratatui-powered terminal interface. Browse your experiments with zero latency and full visual feedback.

Safe Cleanup

Delete old workspaces directly from the interface with Ctrl+D. Includes a safety confirmation popup.

Editor Launch

Open your favorite IDE (VSCode, Neovim, Helix) instantly in the selected workspace using the Ctrl+E shortcut.

Built-in Themes

Switch between popular color schemes like Catppuccin, Gruvbox, or Tokyo Night at runtime with Ctrl+T.

Dated Hierarchy

Automatically organizes folders using a YYYY-MM-DD-name format, making chronological navigation effortless.

Cross-platform

Native support for Linux, macOS, and Windows. Uses OS-standard configuration paths for a native feel everywhere.

Configuration

Customize your experience by creating a config.toml file. The application checks for configuration in the following order:

Linux
1. OS Default ~/.config/try-rs/config.toml
2. Home Local ~/.config/try-rs/config.toml
macOS
1. OS Default ~/Library/Application Support/try-rs/config.toml
2. Home Local ~/.config/try-rs/config.toml
Windows
1. OS Default %AppData%\try-rs\config.toml
2. Home Local ~/.config/try-rs/config.toml

Config File Example

# config.toml
tries_path = "~/Development/playground"
editor = "code" # Optional: code, nvim, hx, etc.

[colors]
title_try = "Magenta"
title_rs = "White"
search_box = "Green"
list_date = "Blue"
list_highlight_bg = "Magenta"
list_highlight_fg = "Black"
help_text = "DarkGray"
status_message = "Yellow"
popup_bg = "DarkGray"
popup_text = "LightRed"

Environment Variables

TRY_PATH Path

Overrides the path where experiments are stored.

export TRY_PATH="/home/user/tmp"
TRY_CONFIG_DIR Directory

Overrides the default configuration directory.

export TRY_CONFIG_DIR="/home/user/.myconfig"
TRY_CONFIG Filename

Overrides the config filename (defaults to config.toml).

export TRY_CONFIG="custom.toml"
